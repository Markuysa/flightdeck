Prove the top-level acceptance criteria (PRD §6) end to end against a fixture project,
with dispatch and merge mocked at the boundary.

## Likely files
- `internal/e2e/{e2e_test.go,fixtures}` (or `ui/e2e/` if driven through the built UI)

## Acceptance criteria
- [ ] Boots the server against a fixture repo with tickets in every status and asserts the
      board derives each one correctly over the real HTTP API.
- [ ] Asserts a lying `status: ready` on a blocked ticket still renders blocked (the
      ADR-001 guarantee) through the full stack, not just the derive unit test.
- [ ] Dispatch is disabled for a non-ready ticket and, when fired, calls the (stubbed)
      routine endpoint with the right payload; approve merges only the named PR via a
      stubbed GitHub.
- [ ] Asserts no token-shaped string is ever sent to the client across the covered flows.
- [ ] Deterministic — fixtures and stubs only, no live services. Runs in CI.
